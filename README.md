# Demo 1: CLI 工程化基座(分步骤学习)

**所属阶段**:[阶段一:CLI 工程化基座](../前端工程化_AI_CLI_学习路线.md)

**学习方式**:跟着 [`学习指南.md`](./学习指南.md) 一步步做,不直接看完整代码。

## 为什么分步骤

工业级 CLI 基座涉及 `tsc` / `tsup` / `vitest` / `commander` / `changesets` 等一堆工具,一次性配齐会让人懵。

这份指南拆成 8 个小步骤,**每步只引入一个新概念**,跑通了再进下一步。慢就是快。

## 学习节奏

| Step | 做什么 | 引入概念 | 预计耗时 |
|------|--------|---------|---------|
| 0 | 准备目录 | `npm init` | 5 分钟 |
| 1 | 写第一个 TS 文件 | `tsc`(TypeScript 编译器) | 30 分钟 |
| 2 | 直接跑 TS | `tsx`(免编译运行) | 15 分钟 |
| 3 | 做 CLI 命令 | `commander` | 45 分钟 |
| 4 | 加构建打包 | `tsup` + ESM/CJS | 1 小时 |
| 5 | 写第一个测试 | `vitest` | 45 分钟 |
| 6 | 加 TS 严格模式 | `tsconfig` 严格选项 | 45 分钟 |
| 7 | 加覆盖率 | `vitest coverage` | 30 分钟 |
| 8 | (可选)CI + 发布 | `changesets` + GitHub Actions | 1 小时 |

总计 5-6 小时,可以分 2-3 天完成。

## 验收标准(最终达成)

跑完所有步骤后,你应该有一个:
- [ ] 用 TypeScript 写的 CLI,能 `npx` 运行
- [ ] 有单元测试 + 集成测试
- [ ] 覆盖率 ≥ 80%
- [ ] 能 build 出 `dist/` 产物
- [ ] (可选)发布到 npm

## 开始

打开 [`学习指南.md`](./学习指南.md),从 Step 0 开始。

- 一步做完,验证通过,再进下一步
- 卡壳了回头看那一步的"常见错误"
